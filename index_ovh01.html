🧮 1. Le compute

A quoi ça sert ?

  • Lancer et gérer des serveurs virtuels (appelés "instances") dans le cloud
  • Ces serveurs sont disponibles à la demande : tu décides du nombre, de la taille, et du type selon tes besoins, et c’est OVH qui s’occupe de l’infrastructure

Exemples d’utilisation : héberger un site web, une API, des applications, effectuer des calculs, lancer des scripts en ligne

Similaire à : les “machines virtuelles” chez d’autres cloud (AWS EC2, Azure VM)

Utilité ? Oui, pour héberger le conteneur Docker


📦 2. Storage & backup

A quoi ça sert ?

  • Proposer différents types de stockage pour tes fichiers, données, sauvegardes et bases de données
  • Adapté selon ton usage : stockage objet (beaucoup de petits fichiers), bloc (disques virtuels pour serveurs), sauvegarde automatisée, etc

Exemples d’utilisation : stocker des fichiers volumineux, des sauvegardes d’applications, créer des bases de données performantes, archiver des documents

Similaire à : AWS S3, Google Cloud Storage, Azure Blob Storage

Utilité ? Oui, pour les FEC et fichiers des stocks uploadés, les fichiers Excel générés


🔧 3. Network

A quoi ça sert ?

  • Connecter tes ressources cloud OVH entre elles et avec d’autres produits (ex : serveurs dédiés, CDN, réseau privé)
  • Gestion des adresses IP, VPN, firewalls, sécurité réseau

Exemples d’utilisation : relier tes serveurs cloud via un réseau privé, sécuriser l’accès avec des règles de firewall, utiliser un VPN entre des infrastructures

Similaire à : VPC (Virtual Private Cloud), VPN, CDN chez d’autres cloud

Utilité ? Oui, pour rendre l’accès payant et restreint via IP, il faut :

  • configurer le réseau, les règles de firewall (n’autoriser que certaines IPs)
  • protéger le serveur contre les attaques et l’accès public non autorisé

⚙️ 4. Containers & Orchestration

A quoi ça sert ?

  • Lancer, gérer et automatiser des conteneurs Docker (et équivalents) dans le cloud
  • Orchestration (= gestion automatisée, déploiement, montée/descente en charge) via des outils comme Kubernetes

Exemples d’utilisation : déployer rapidement des applications, services, microservices dans des conteneurs ; automatiser la gestion des ressources ; scaler facilement

Similaire à : AWS ECS/EKS, Azure AKS, Google Kubernetes Engine

Utilité ? Oui, seulement si déploiement de plusieurs instances ou ajout d'autres services dockerisés


🎁 5. Databases & Analytics

A quoi ça sert ?

  • Proposer des bases de données (MySQL, PostgreSQL, MongoDB…) comme service cloud, avec sauvegarde, scalabilité, haute disponibilité
  • Fournir des outils Big Data et d’analyse de données (Hadoop, Spark, managed analytics)

Exemples d’utilisation : stocker de gros volumes de données, analyser en temps réel, traiter du “Big Data”, dashboard, BI

Similaire à : AWS RDS, Google BigQuery, Azure SQL Database, AWS EMR

Utilité ? Non


💻 6. AI & Machine Learning

A quoi ça sert ?

  • Mettre à disposition des infrastructures optimisées (GPU, CPU puissants), plateformes et outils pour l’intelligence artificielle et l’entraînement de modèles de machine learning

Exemples d’utilisation : entraîner des modèles IA, faire du traitement d’image, NLP, automatiser des analyses grâce au machine learning directement dans le cloud

Similaire à : AWS SageMaker, Azure Machine Learning, Google AI Platform

Utilité ? Non sauf ajout IA future
